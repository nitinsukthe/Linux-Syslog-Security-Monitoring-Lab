# Linux Syslog Security Monitoring Lab

## Project Overview

This project demonstrates hands-on Linux log analysis and security monitoring using Ubuntu and the rsyslog logging framework. The objective was to understand how Linux systems generate, store, and manage log data while applying security analysis techniques used by SOC Analysts and Cloud Security Engineers.

The project focused on reviewing syslog configurations, analyzing system logs, investigating authentication events, detecting failed login attempts, monitoring privileged account activity, and summarizing log data using native Linux command-line tools.

Through this project, real-world security monitoring techniques were applied to identify authentication failures, sudo activity, root access events, and system processes recorded within Linux log files.

---

## Objectives

- Understand Linux syslog architecture
- Review rsyslog configuration settings
- Access and analyze Linux log files
- Investigate authentication-related events
- Detect failed login attempts
- Monitor privileged account usage
- Analyze sudo activity
- Summarize log data using Linux commands
- Map security findings to MITRE ATT&CK
